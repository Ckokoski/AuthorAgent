- **Source-IP allowlist (`AUTHORCLAW_ALLOWED_IPS`)** (security review; new item raised during CORS work) — a network-level gate, distinct from CORS, restricting which *client machines* may reach the server at all (applies to every client, in front of even the bearer token). *Investigation first surfaced a make-or-break caveat: the current Docker deploy (default bridge + published port, no `daemon.json` so `userland-proxy` defaults true) makes the container see the bridge gateway IP for every external client, so an app-level per-IP gate only sees real IPs under host-net/native/VPS or behind a trusted proxy — the most robust enforcement for the VPS "only my home IP" case is the host firewall / provider security group, with this as defense-in-depth. Decisions (confirmed by user): **unset = allow all + a startup notice** (enforcement strictly opt-in; auto-deriving a local /24 was rejected as a trap — in Docker it derives the 172.x container subnet, on a VPS the datacenter subnet, blocking the real client both ways); **always allow loopback** as a recovery path; **opt-in `AUTHORCLAW_TRUST_PROXY=1`** reads the client IP from `X-Forwarded-For` for reverse-proxy deploys (off by default; spoofable unless behind a sole-ingress proxy). Implementation (`gateway/src/index.ts`): added `ipaddr.js` as a direct dep (was transitive via `proxy-addr`) for correct IPv4/IPv6/IPv4-mapped + CIDR matching; constructor parses `AUTHORCLAW_ALLOWED_IPS` (comma-separated IPs/CIDRs; single IPs → /32 or /128; invalid entries warned and skipped), sets `trust proxy`, and registers an Express gate (403 + audit `ip_blocked` on deny) ahead of the auth middleware; a matching `io.use()` gates the Socket.IO handshake; `isIpAllowed()` uses `ipaddr.process()` to normalize IPv4-mapped addresses and treats `range()==='loopback'` as always-allowed; startup logs the posture (`ℹ IP allowlist: not set …` / `✓ IP allowlist: N rule(s) enforced …`). Smoke test Phase 4 (trust-proxy on, so allow/deny is deterministic from loopback via XFF): exact IP allowed, CIDR member allowed, unlisted IP → 403 even with a valid token (proves the gate is in front of auth), loopback always allowed, and the enforcement log present. 16 checks total pass; `npx tsc --noEmit` clean. Next security item: Helmet CSP.*
